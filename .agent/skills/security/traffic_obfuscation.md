# 🌫 Traffic Obfuscation & Censorship Evasion

Этот skill содержит информацию и инструкции по обфускации (маскировке) трафика для обхода глубокого анализа пакетов (DPI) интернет-провайдеров и ТСПУ (Технических средств противодействия угрозам).

## 🕶 1. XTLS-Reality (VLESS)

Reality исключает использование сертификатов на стороне сервера, выполняя перенаправление (handshake hijacking) к известному разрешенному сайту (например, `www.microsoft.com` или `www.apple.com`).

### Критические настройки Reality:
* **SNI (Server Name Indication):** Должен быть крупным, популярным ресурсом за пределами юрисдикции блокировок, поддерживающим HTTP/2 или HTTP/3 и TLS v1.3.
* **Short ID:** Должен регулярно меняться или генерироваться случайным образом (16-символьная HEX строка).
* **Fingerprint (UTLS):** Клиент должен имитировать реальный современный браузер (`chrome`, `firefox`, `safari`, `edge`). Запрещено использовать пустой или стандартный TLS-отпечаток (это выдает сигнатуру Go-клиента Xray).

## ⚡ 2. Hysteria 2 (UDP-based Obfuscation)

Hysteria 2 использует модифицированный протокол QUIC (поверх UDP), который оптимизирован для нестабильных сетей и имеет встроенный обфускатор для маскировки структуры QUIC-пакетов под случайный шум.

### Настройка обфускации Salamander:
В конфигурационном файле Hysteria (`/etc/hysteria/config.yaml`):
```yaml
# Порт прослушивания (часто маскируется под 443/UDP)
listen: :10443

# Использование обфускации типа salamander
obfuscator:
  type: salamander
  password: SalamanderObfs2026SecretKey
```
*На стороне клиента обязательно указывается тот же тип обфускатора и секретный пароль.*

## 🧬 3. ShadowTLS

Позволяет оборачивать неподдерживаемые протоколы (например, Shadowsocks или обычный Trojan) в TLS-сессию, которая полностью копирует рукопожатие (TLS Handshake) легитимного сервера.

* Используется схема: `Клиент` ➔ `ShadowTLS Клиент` ➔ `DPI (видит чистый TLS до Apple)` ➔ `ShadowTLS Сервер` ➔ `Локальный Shadowsocks`.
* Подделывает аутентификацию на основе общих секретов, пересылая оригинальные пакеты TLS-handshake от целевого сайта.

## 🛡 4. AmneziaWG (Модифицированный WireGuard)

Классический WireGuard легко блокируется, так как имеет жестко заданные заголовки и размеры пакетов первого обмена (Initiation/Response). AmneziaWG меняет заголовки на случайные байты и добавляет случайный мусор (Padding) в начало пакетов.

### Ключевые параметры AmneziaWG (в секции `[Interface]`):
```ini
[Interface]
# Замена стандартных магических чисел заголовков пакетов
Jc = 4
Jmin = 5
Jmax = 1000
S1 = 15
S2 = 23
H1 = 1
H2 = 2
H3 = 3
H4 = 4
```
Если параметры на сервере и клиенте не совпадают, сервер проигнорирует подключение.
