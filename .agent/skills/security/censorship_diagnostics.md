# 🕵️‍♂️ IP & Censorship Diagnostics

Этот skill содержит сценарии проверки блокировок IP-адресов серверов и конкретных протоколов государственными системами фильтрации (ТСПУ) или отдельными провайдерами.

---

## 🔎 Признаки блокировки (Block Scenarios)

1. **Блокировка по IP (Полный бан):** 
   * Сервер не отвечает ни на `ping`, ни на `ssh`, ни на `VLESS/Hysteria` порты изнутри страны.
   * При этом сервер доступен при подключении через сторонний (например, зарубежный) VPN.
2. **Блокировка по сигнатуре протокола (DPI):**
   * SSH работает отлично.
   * Подключение по VLESS Reality зависает на этапе TLS-handshake (клиент отправляет Client Hello, но серверный ответ Server Hello сбрасывается провайдером).
3. **Блокировка по SNI:**
   * Подключения к серверу проходят, если указать неблокируемый SNI (например, `google.com`), но сбрасываются при обращении к запрещенным доменам.

## 🛠 Методы диагностики (How to Test)

### 1. Проверка доступности TCP-портов без пинга (через TCP SYN)
Так как обычный ICMP-пинг часто отключен в облаке или заблокирован провайдером, используйте `curl` или `nc` для проверки открытого порта:

```bash
# Быстрая проверка TCP порта 443
curl -vI https://37.1.212.51:443 --connect-timeout 5

# Тестирование порта через netcat
nc -zvw3 37.1.212.51 443
```
*Если в ответе пишет `Connection timed out` только из РФ — порт или IP заблокирован.*

### 2. Скрипт проверки «Жив ли IP» из разных точек
Если у вас есть доступ к серверам в разных сетях (например, Yandex VM и локальный компьютер):
```bash
# Запустить на Yandex VM (из РФ)
curl -s -o /dev/null -w "%{http_code}" https://37.1.212.51:8085/admin/ui --connect-timeout 3
# Если возвращает 200/405 — веб-сервер панели доступен напрямую.
```

### 3. Логирование на стороне Xray
Если клиент не может подключиться, проверьте лог сервера на предмет входящих пакетов:
```bash
# Смотрим попытки соединений
journalctl -u xray -n 100 --no-pager | grep -i -E "accepted|rejected"
```
*Если при попытке подключения в логах сервера тишина — пакеты клиента блокируются провайдером на пути к серверу.*

## 🆘 Действия при блокировке IP

1. **Временное переключение на CDN (Cloudflare):**
   * Маршрутизировать трафик клиентов через прокси Cloudflare (это спасет от блокировки IP, но увеличит пинг).
2. **Замена IP-адреса:**
   * Запросить новый IP у хостинга (многие VPS-хостинги меняют IP бесплатно или за символическую плату).
3. **Использование Релея (Relay) с чистым IP:**
   * Перенаправить трафик клиентов через промежуточную виртуальную машину (например, в Yandex Cloud), IP которой еще не попал в бан-лист.
4. **Смена портов:**
   * Перенести Xray с `443` порта на менее популярный (например, `8443`, `2053`, `2083`).
