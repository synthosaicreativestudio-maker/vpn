## 1. Как работает фильтрация

```
Пакет → [L3: IP в белом списке?]
         НЕТ → DROP (пакет исчезает)
         ДА  → [L7: SNI в чёрном списке?]
                ДА → RST (соединение рвётся)
                НЕТ → PASS
```

- **63 126 IP** в белом списке из 46 млн российских (0.14%)
- TCP 80/443/22 работают на whitelisted IP
- **UDP блокируется полностью** (QUIC, DNS, WireGuard, Hysteria2)
- Затрагивает **мобильный интернет** (домашние провайдеры пока нет)
- Работает неравномерно по регионам и провайдерам

## 5. Cloudflare — полный анализ

### 5.1 IP Cloudflare (проверено 11.05.2026)

| Сервис | IP | Подсеть | В белом списке? |
|--------|-----|---------|----------------|
| CDN | `104.16.x.x` | `104.16.0.0/12` | ❌ **НЕТ** — зарубежная ASN |
| Workers | `8.47.69.0` | Cloudflare | ❌ **НЕТ** |
| WARP | `162.159.192.1` | Cloudflare | ❌ **НЕТ** + UDP заблокирован |
| Tunnel | `198.41.192.x` | Cloudflare | ❌ **НЕТ** |

### 5.2 Вердикт по каждому сервису Cloudflare

#### ❌ Cloudflare WARP
- Использует **WireGuard (UDP)** — UDP полностью блокируется при БС
- IP не в белом списке
- В "обычном" режиме тоже нестабилен в РФ (блокируется ТСПУ)
- **Вердикт: БЕСПОЛЕЗЕН при белых списках**

#### ❌ Cloudflare Workers
- IP Workers (`8.x.x.x`) не в белом списке
- В некоторых регионах Workers уже заблокированы в РФ
- Даже если бы работали — это HTTP-прокси, не VPN-туннель
- **Вердикт: НЕ РАБОТАЕТ при белых списках**

#### ❌ Cloudflare CDN (VLESS+WS через CF)
Схема: `Клиент → CF CDN (104.16.x.x) → WS → наш VPS → Интернет`

- IP CDN (`104.16.0.0/12`) **не в белом списке** → DROP на L3
- РКН рекомендовал отказаться от CF, TLS ECH блокируется
- Hiddify и Happ поддерживают VLESS+WS+CDN, **но при БС CDN недоступен**
- **Вердикт: НЕ РАБОТАЕТ при белых списках**

#### ❌ Cloudflare Tunnel (Argo)
- IP тоннелей (`198.41.192.x`) не в белом списке
- Активно блокируется ТСПУ
- **Вердикт: НЕ РАБОТАЕТ при белых списках**

### 5.3 Единственный сценарий где CF полезен

**Cloudflare CDN работает НА ДОМАШНЕМ интернете** (без БС). Схема:

```
Домашний Wi-Fi → CF CDN → VLESS+WS → наш US VPS → Интернет
```

Это даёт скрытие IP нашего сервера за CDN. Полезно **не для обхода БС**, а для защиты от блокировки IP нашего US-сервера на домашних провайдерах.

| Hysteria2 | ✅ | ✅ (но UDP!) |

### Для relay-chain (обход БС)

Клиенты подключаются к **relay VPS в РФ** по:
- `VLESS+Reality+Vision` на порт 443 — **оба поддерживают** ✅
- SNI: `ozon.ru` / `wildberries.ru` — прозрачно для клиента

Подписка будет содержать **оба набора**:
```
# Прямые (Wi-Fi / домашний):
vless://uuid@37.1.212.51:443?...#🔌 Vision (Прямой US)

# Через relay (мобильный / БС):
vless://uuid@RELAY_IP:443?sni=ozon.ru&...#📡 Vision (Relay РФ)
```

Hiddify/Happ автоматически выберут рабочий сервер.

## 9. Ёмкость relay: сколько пользователей

- **1 vCPU / 1 GB RAM** → 5-10 пользователей комфортно
- Xray потребляет ~150-300 MB RAM
- **Лимит не ресурсы, а заметность** — аномальный трафик с одного IP = риск
- Для личного использования + семья — **одного VPS достаточно**
