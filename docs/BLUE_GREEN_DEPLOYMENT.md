# 🔵🟢 Регламент безопасного развертывания (Blue-Green Deployment)

Для предотвращения сбоев в работе VPN, ошибок ИИ при редактировании кода и минимизации времени простоя (Downtime) в проекте применяется концепция **Blue-Green** развертывания для панели управления и **предварительное тестирование** для ядра Xray.

---

## 1. Архитектура сред на сервере

На сервере США (`37.1.212.51`) настраиваются две параллельные папки с кодом проекта:

*   **Blue (Синяя среда):** Папка `/root/vpn-blue`. Служба `vpn-panel-blue.service` работает на порту `8085`.
*   **Green (Зеленая среда):** Папка `/root/vpn-green`. Служба `vpn-panel-green.service` работает на порту `8086`.

### Общие ресурсы (Синглтоны)
База данных SQLite должна быть общей, чтобы при переключении сред не терялись пользователи и транзакции.
*   Путь к БД: `/var/lib/vpn/data/panel.db` (обе среды ссылаются на этот путь в своих файлах `.env`).
*   Путь к локальным гео-файлам: `/var/lib/vpn/geo/` (общая папка для раздачи).

---

## 2. Схема переключения трафика через Caddy

Caddy выступает в роли балансировщика. Переключение сред происходит мгновенно путем изменения порта апстрима в `Caddyfile` и его перезагрузки (`systemctl reload caddy`), что происходит без разрыва активных соединений.

### Шаблон Caddyfile (активна Blue):
```caddy
:80 {
    handle /sub/* {
        reverse_proxy 127.0.0.1:8085  # Порт Blue
    }
    handle /routing/* {
        reverse_proxy 127.0.0.1:8085  # Порт Blue
    }
    ...
}
```

### Шаблон Caddyfile (активна Green):
```caddy
:80 {
    handle /sub/* {
        reverse_proxy 127.0.0.1:8086  # Порт Green
    }
    handle /routing/* {
        reverse_proxy 127.0.0.1:8086  # Порт Green
    }
    ...
}
```

## 3. Пошаговый алгоритм обновления (Деплой)

Когда необходимо внести изменения в код панели или бота:

### Шаг 1. Определение неактивной среды
Проверьте текущую активную среду в Caddyfile:
```bash
cat /etc/caddy/Caddyfile | grep reverse_proxy
```
*Если активен порт 8085 (Blue), значит целевая для деплоя среда — Green (порт 8086), и наоборот.*

### Шаг 2. Внесение изменений в неактивную среду
Выгрузите обновленный код в папку неактивной среды (например, `/root/vpn-green`).

### Шаг 3. Запуск и верификация неактивной среды
Запустите неактивную службу и проверьте ее статус:
```bash
systemctl restart vpn-panel-green
systemctl status vpn-panel-green
```
Выполните тестовый запрос для проверки работоспособности (Health Check):
```bash
curl -f http://127.0.0.1:8086/health
```
*Запрос должен вернуть HTTP 200 OK. Если возвращается ошибка — деплой останавливается, изменения исправляются в тестовой среде.*

### Шаг 4. Переключение трафика (Switch)
Измените порт в `/etc/caddy/Caddyfile` на порт новой среды и обновите Caddy:
```bash
systemctl reload caddy
```
*С этого момента 100% новых запросов пользователей идут на новую версию.*

### Шаг 5. Остановка старой среды
Убедившись, что новая среда работает стабильно, остановите старую службу:
```bash
systemctl stop vpn-panel-blue
```

## 4. Безопасное изменение конфигурации Xray

Ядро Xray не может быть запущено в двух экземплярах на одном порту (например, 443). Поэтому для него применяется метод **тестирования конфигурации перед запуском**.

### Алгоритм изменения конфига Xray:

1.  **Создание резервной копии:**
    ```bash
    cp /etc/xray/config.json /etc/xray/config.json.backup
    ```
2.  **Редактирование тестового файла:**
    Вносятся правки в `/etc/xray/config.json.test`.
3.  **Валидация синтаксиса и структуры (Критический шаг):**
    Запускается встроенный валидатор Xray:
    ```bash
    xray test -config /etc/xray/config.json.test
    ```
    *Если валидатор выдает ошибки — файл не применяется, связь с пользователями не нарушается.*
4.  **Применение конфигурации:**
    Если тест прошел успешно:
    ```bash
    cp /etc/xray/config.json.test /etc/xray/config.json
    systemctl restart xray
    ```
5.  **Экстренный откат (Rollback):**
    Если после перезапуска статус службы не `active` или пользователи сообщают о проблемах:
    ```bash
    cp /etc/xray/config.json.backup /etc/xray/config.json
    systemctl restart xray
    ```
    *Связь восстанавливается до исходного рабочего состояния за 2 секунды.*
