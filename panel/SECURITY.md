# 🛡️ Server Security — Hardening Report

Сервер `37.1.212.51` был полностью защищен по стандартам Марта 2026.

## 🔐 SSH: Ключи и доступы
- **Метод входа**: Только по SSH-ключам (Ed25519).
- **Пароли**: Вход по паролю **полностью отключен** на уровне `sshd_config`.
- **Root login**: Прямой вход под рутом по паролю запрещен (`prohibit-password`).
- **Брутфорс**: Ограничение попыток входа до 3 (`MaxAuthTries 3`).

### Как добавить новый SSH-ключ:
1.  Сгенерируйте ключ на локальной машине: `ssh-keygen -t ed25519`
2.  Добавьте его на сервер (пока у вас есть доступ): `ssh-copy-id root@37.1.212.51`

## 🧱 Firewall (UFW)
Firewall работает в режиме **Whitelist** (запрещено всё, кроме разрешенного).

### Открытые порты:
- `22/tcp`: SSH (включен `LIMIT` — защита от перебора).
- `443/tcp`: Shadow-TLS (основной канал).
- `10443/tcp`: Xray Vision.
- `10443/udp`: Hysteria 2.
- `10444/tcp`: Xray xHTTP.
- `18443/tcp`: Xray gRPC.
- `30445/udp`: TUIC.
- `8443/tcp`: Caddy (Marzban dashboard).

### Закрытые порты:
- `8000`: Marzban (доступен только через локальный прокси или Caddy).
- `8080`: TinyProxy (теперь слушает только `127.0.0.1`).

## 🛡️ Ядро и Система
- **RP Filter**: Включен (`net.ipv4.conf.all.rp_filter = 1`) для предотвращения IP spoofing.
- **Updates**: Включена служба `unattended-upgrades` для автоматической установки патчей безопасности.
- **Users**: У системного пользователя `hysteria` удален доступ к командной строке (`nologin`).

## 🚦 Рекомендации
1.  **Reboot**: После последнего обновления ядра рекомендуется выполнить перезагрузку сервера (`reboot`).
2.  **Backup**: Регулярно делайте копии БД панели (`panel/data/panel.db`).
3.  **Logs**: Проверяйте попытки входа в `/var/log/auth.log` и бан-листы `fail2ban-client status`.
